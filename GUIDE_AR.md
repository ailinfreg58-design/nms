# دليل استخدام نظام مراقبة الشبكات NMS

## نظرة عامة
نظام متكامل لمراقبة وإدارة أبراج الاتصالات والأجهزة الشبكية مع نظام إدارة التذاكر والتنبيهات الفورية.

---

## المتطلبات الأساسية

### الأجهزة
- معالج: Intel/AMD (أي نوع)
- الذاكرة: 512 MB على الأقل
- التخزين: 500 MB

### البرامج
- Node.js 18+
- npm / pnpm / yarn
- متصفح حديث (Chrome, Firefox, Safari, Edge)

### الخدمات الخارجية
- Neon PostgreSQL database
- Better Auth for authentication

---

## التثبيت والإعداد

### 1. استنساخ المشروع
```bash
git clone <repository-url>
cd nms-project
```

### 2. تثبيت المتطلبات
```bash
pnpm install
# أو
npm install
```

### 3. إعداد متغيرات البيئة
انسخ `.env.example` إلى `.env.local`:
```bash
cp .env.example .env.local
```

ثم عدّل القيم:
```env
# قاعدة البيانات
DATABASE_URL=postgresql://user:password@host:port/database

# المصادقة
BETTER_AUTH_SECRET=<32-character-random-string>
BETTER_AUTH_URL=http://localhost:3000

# بيئة التطوير
NODE_ENV=development
```

**لإنشاء BETTER_AUTH_SECRET آمن:**
```bash
openssl rand -base64 32
```

### 4. تشغيل الخادم
```bash
pnpm dev
```

ستظهر الرسالة:
```
> ready - started server on 0.0.0.0:3000, url: http://localhost:3000
```

## استخدام النظام

### الخطوة 1: إنشاء حساب
1. انتقل إلى `http://localhost:3000/sign-up`
2. أدخل:
   - الاسم الكامل
   - البريد الإلكتروني
   - كلمة المرور (8 أحرف على الأقل)
3. اضغط "Sign Up"

### الخطوة 2: تسجيل الدخول
1. انتقل إلى `http://localhost:3000/sign-in`
2. أدخل بريدك الإلكتروني وكلمة المرور
3. اضغط "Sign In"

### الخطوة 3: إضافة برج (Tower)
1. من لوحة المعلومات، اضغط على "Towers"
2. اضغط "+ Add Tower"
3. ملء البيانات:
   - **Tower Name**: مثل "Tower A1"
   - **Code**: مثل "T-001"
   - **Region**: "North Region"
   - **Latitude**: 24.860735
   - **Longitude**: 46.659385
4. اضغط "Create Tower"

### الخطوة 4: إضافة أجهزة (Devices)
1. من لوحة المعلومات، اضغط على "Devices"
2. اضغط "+ Add Device"
3. ملء البيانات:
   - **Tower**: اختر البرج الذي أنشأته
   - **Device Type**: Camera / Router / Inverter / LPU
   - **Device Name**: مثل "Main Camera"
   - **IP Address**: 192.168.1.100
   - **MAC Address**: (اختياري)
4. اضغط "Add Device"

### الخطوة 5: مراقبة الأجهزة
- يتم الـ Ping تلقائياً كل 5 دقائق
- تحديث حالة الأجهزة فوراً
- عرض Response Time لكل جهاز

### الخطوة 6: إنشاء التذاكر
1. من لوحة المعلومات، اضغط على "Tickets"
2. اضغط "+ New Ticket"
3. ملء البيانات:
   - **Device**: اختر الجهاز المتعلق
   - **Priority**: Low / Medium / High / Critical
   - **Title**: عنوان المشكلة
   - **Description**: وصف مفصل
4. اضغط "Create Ticket"

### الخطوة 7: إدارة التذاكر
- تغيير حالة التذكرة: Open → In Progress → Resolved → Closed
- تصفية حسب الحالة والأولوية

## الإحصائيات والمراقبة

### لوحة المعلومات توعرض:
- **إجمالي الأجهزة**: العدد الكلي للأجهزة المراقبة
- **أجهزة في الخدمة (Online)**: الأجهزة الفعالة الآن
- **أجهزة معطلة (Offline)**: الأجهزة التي لا تستجيب
- **نسبة التشغيل (Uptime)**: النسبة المئوية للأجهزة الفعالة

### إحصائيات الأبراج:
- عدد الأجهزة في كل برج
- عدد الأجهزة الفعالة
- نسبة التشغيل لكل برج

### إحصائيات التذاكر:
- عدد التذاكر المفتوحة
- عدد التذاكر الحرجة
- حالة التذاكر المختلفة

## المراقبة التلقائية

### نظام الـ Ping
- يعمل تلقائياً كل **5 دقائق**
- يختبر **جميع الأجهزة** في النظام
- يسجل **Response Time** لكل جهاز
- ينشئ **سجل تاريخي** للمراقبة

### التنبيهات الفورية
- **جهاز معطل**: ينشئ تنبيه فوري
- **Uptime عالي**: ينشئ تذكرة تلقائية
- **Response Time عالي**: إنذار تحذيري

## قاعدة البيانات

### الجداول الرئيسية
| الجدول | الوصف |
|--------|-------|
| user | بيانات المستخدمين |
| tower | بيانات الأبراج |
| device | بيانات الأجهزة |
| ping_log | سجلات الـ Ping |
| ticket | التذاكر الفنية |

- كل جهاز قد يكون له تذاكر (tickets)
- كل تذكرة ترتبط بجهاز واحد

## معالجة الأخطاء الشائعة

### خطأ: Database connection refused
**السبب**: قاعدة البيانات غير متصلة
**الحل**:
```bash
# تحقق من DATABASE_URL
echo $DATABASE_URL

# أعد تعيين الاتصال
pnpm dev
```

### خطأ: BETTER_AUTH_SECRET not set
**السبب**: مفتاح المصادقة لم يتم تعيينه
**الحل**:
```bash
# أنشئ مفتاح جديد
openssl rand -base64 32

# أضفه في .env.local
BETTER_AUTH_SECRET=<الناتج>
```

### خطأ: Port 3000 already in use
**السبب**: المنفذ مستخدم من قبل برنامج آخر
**الحل**:
```bash
# غيّر المنفذ
PORT=3001 pnpm dev
```

### الأجهزة تظهر offline دائماً
**السبب**: قد لا تكون عناوين IP صحيحة أو firewall
**الحل**:
1. تحقق من عناوين IP
2. تأكد من أن الأجهزة متاحة
3. تحقق من قواعد الـ firewall

## الدعم والمساعدة

### الأسئلة الشائعة (FAQ)
**س: كم تكون مدة المراقبة التلقائية؟**
ج: كل 5 دقائق بشكل آلي

**س: هل يمكن تغيير مدة المراقبة؟**
ج: نعم، عدّل `node-cron` في `lib/background-jobs.ts`

**س: كيف أحذف برج؟**
ج: انتقل للأبراج واختر الحذف (سيحذف الأجهزة أيضاً)

**س: هل يمكن تصدير البيانات؟**
ج: يمكنك الاتصال مباشرة بقاعدة البيانات عبر Neon
